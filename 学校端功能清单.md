# 学平险 · 学校端功能清单 PRD

> **版本**：v1.0 · 2026-05-25  
> **原型入口**：[`学平险-学校端--Html原型/index.html`](../../学平险-学校端--Html原型/index.html)  
> **演示租户**：立德中学（`school_id`：SCH_00128）

---

## 1. 产品定位

学校工作人员在本端完成：

- 学生档案维护与关爱标签管理  
- 按学期查看家长端投保订单汇总（班级投保名单）  
- 投保二维码生成与分发  
- 校内风险事件登记与安全教育任务跟进  
- 校内组织、成员与操作审计（校级管理员）

**不在学校端维护**：渠道产品、区域上架、本校方案定价（由平台端配置后下发）。

### 2.2 角色与数据范围（MVP）

| 角色 | 说明 | 数据范围（演示口径） |
|------|------|----------------------|
| 校级管理员 `admin` | 全校配置与审计 | 6 年级 12 班、全部学生 |
| 年级组长 `grade` | 所辖年级业务 | 演示为**高一年级** 3 个班 |
| 班主任 `class` | 所辖班级业务 | 演示为**高一(3)班**； |

同一账号可拥有多个身份（如年级组长 + 班主任），登录时根据当前身份进行展示菜单权限与数据权限，学生档案、班级投保名单根据账号权限范围展示。

## 3. 全局业务规则

### 3.1 统计学期

| 规则 | 说明 |
|------|------|
| 春季学期 | 投保日期在 **2–8 月** → `YYYY 春季`；保障期示例：2/1–8/31 |
| 秋季学期 | 投保日期在 **9–12 月** → `YYYY 秋季`；保障期示例：9/1–次年 1/31 |
| 默认学期 | 由系统参考当前日期自动匹配（演示参考日 `2026-06-01` → **2026 春季**） |
| 可选学期 | 2025春季/2025秋季/2026 春季 / 2026秋季 /2027春季/2027秋季（去年、今年、明年） |

**本学期已投保**口径：家长端投保**订单流水**，按 `班级 + 学生姓名` 去重计数（与学籍号无关）。

### 3.2 核心数据关系

```
组织与成员（年级/班级）
    ↓
学生档案 ←—— 姓名+班级 ——→ 班级投保名单（投保订单流水）
    ↓                              ↑
风险事件登记                    投保二维码（家长扫码投保）
    ↓
安全教育任务 ← 平台下发
```

| 关系 | 规则 |
|------|------|
| 投保订单 ↔ 学生档案 | **相互独立**；仅通过 **姓名 + 班级** 弱关联展示 |
| 未建档也可投保 | 订单计入「已投保人数」，档案关联显示「未建档」 |
| 删除学生档案 | 档案不可恢复；历史投保订单、风险事件**保留**，但不再关联该档案 |
| 数据同步 | 看板文案：约 **15 分钟**同步一次（正式需后端 SLA） |

### 3.3 保障状态（班级投保名单详情）

相对系统参考日期判断：

| 状态 | 条件 |
|------|------|
| 保障中 | 当前日期在保障起止期内 |
| 临近过期 | 距保障止期 **< 30 天** |
| 已失效 | 保障止期已过 |

**必填（6 项）**：姓名、性别、出生日期、学籍号、年级、班级（班级须在组织与成员已建）。

**选填（20 项）**：身份证号、血型、入学日期、户籍、住址、校内紧急联系人、监护人 1/2（关系/姓名/电话/单位/是否主要）、过敏史、慢性病、活动限制、特殊群体标签、健康关注标签。

**导入规则**：学籍号全校唯一，重复按更新；同一单元格多项用英文分号 `;` 分隔。

**删除规则**：档案删除不可恢复；投保订单与风险事件保留，不再关联档案。

## 5. MVP 交付范围汇总

### 5.1 MVP 必做（P0）

1. 登录 + 三角色数据范围  
2. 统计学期口径 + 看板 KPI + 班级投保名单  
3. 学生档案 CRUD + 批量导入 + **删除**  
4. 投保二维码（范围、有效期、海报、打印）  
5. 风险事件登记 + 档案关联 + 结案同步  
6. 安全教育任务查看 + 班级进度  
7. 组织树 + 成员与角色范围（admin）  
8. 投保订单与档案「姓名+班级」关联展示  

### 5.2 MVP 可简化（P1）

- 手机号明文需后端鉴权审计  
- 导入/导出走后端异步任务  
- 二维码短链  
- 安全教育

## 6. 非功能需求（摘要）

| 类别 | 要求 |
|------|------|
| 租户隔离 | 业务数据按 `school_id` 隔离 |
| 权限 | 菜单 + 接口双层校验；系统管理页面前端门禁 + 后端鉴权 |
| 审计 | 档案删除、成员变更、导入任务等写操作日志 |
| 隐私 | 手机号、身份证默认脱敏；明文查看留痕 |
| 性能 | 列表分页；导入单次 ≤1000 行 |
| 同步 | 投保订单至看板/名单：目标 SLA 15 分钟内（待技术确认） |
