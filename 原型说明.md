## 组织与数据

- **区域树（平台端）**：省 → 市 → 学校节点（`region_id`；`平台端-区域管理.html` 维护，含 L2 上架）
- **行政组织（可选/监管）**：可与 `region_id` 绑定，不再作为产品上架主键（原「行政组织树」已合并进区域管理）
- **校内组织树**：学校 → 年级 → 班级（**学校端** `学平险-学校端--组织与成员.html` 维护；见下文）
- **业务数据租户**：`school_id`（仅学校端写入学生/保单等）
- **平台端**：不承载业务明细

## 供给上架（区域树 × listing × 学校绑定）

| 层级 | 页面 | 表意 |
|------|------|------|
| L1 | `平台端-渠道管理.html` | 渠道基础信息：渠道名称、产品品种、机构名称、状态、优惠券/随机立减/订单备注等；**不含定价与上架** |
| L2 | `平台端-区域管理.html` | 全国/省/市区域列表 |
| L2 渠道 | `平台端-区域渠道配置.html?region=` | **列表页**维护上架渠道与区域售价（非弹窗，支持多条） |
| 开通 | `平台端-学校开通.html` | 新增/编辑共用表单：学校名称、所属区域；生成后 `school_id` 不可改；列表「编辑」与详情「编辑学校」同弹窗 |
| L3 | `平台端-学校方案绑定.html` | **渠道配置**：渠道上架与售价、绑定与校定（`?id=` 按校进入） |

**区域树**：`PROVINCE` → `CITY` → `SCHOOL`（学校开通挂载叶子或市下新建校节点）。产品/机构不定义为树子节点类型，而是 **listing 关联 `region_id`**。

**继承**：子区域继承父级已上架渠道，子级可收紧；市级可继承省级区域价或单独定价。

**定价与上架**（同一渠道）：

| 能力 | 维护位置 | 说明 |
|------|----------|------|
| 产品内容 | 渠道管理 | 渠道名称、产品品种、机构名称、状态及优惠券/立减/订单备注规则等 |
| 上架 + 区域价 | 区域管理 | 全国配置上架与参考售价；省/市在上级已上架范围内勾选并填区域售价 |
| 本校定价 | 学校管理 · **渠道配置**（平台维护） | 由平台在学校方案绑定中维护；学校端菜单不提供渠道配置入口 |

**规则**：学校端投保仅展示本校定价（不展示区域价）；须先区域上架且学校已绑定。原 `平台端-区域上架.html`、`平台端-组织树维护.html` 跳转至本页。

**演示**：浙江省挂人保/太保；杭州市收紧人保标准版；立德中学校级试点太保；宁波市停用太保。

## 平台端 · 学校侧角色权限（MVP）

**本期目标**：业务先跑通，不做完整租户 RBAC 平台。

| 步骤 | 平台页面 | 说明 |
|------|----------|------|
| 1 | `平台端-学校开通.html` | **新增学校**时自动挂载平台预置三角色，并**初始化首任校级管理员**（姓名、手机号；初始密码为手机号后 6 位） |
| 2 | `平台端-学校RBAC下发.html` | 查看标准角色；按校查看同步状态；必要时 **重推权限** |
| 3 | 学校端 | `组织与成员` 分配 **角色 + 年级/班级** 后登录办事 |

**MVP 原则**：仅 **校级管理员 / 年级组长 / 班主任** 三个固定角色；学校不新建角色、不维护权限点；本期不做权限点目录、模板编辑、版本发布、本校策略覆盖。

**后续迭代**（`平台端-权限点目录.html`、`平台端-角色模板列表.html` 等跳转说明页）：完整 RBAC 能力。

### 组织与成员 · 页面结构

- **左侧**：年级班级树（本校维护）+「新建年级」「新建班级」（原快捷创建已合并进新建年级）
- **右侧**：成员列表 / 批量导入；工具栏「添加成员」
- **本期**：无校园平台、校务系统对接；结构在学校端创建，不做「同步组织」
- **未来**：对接校务后，可改为「同步为主 + 本校微调」

### 新建年级（含原快捷创建）

| 字段 | 必填 | 说明 |
|------|------|------|
| 年级 | ✓ | 一个下拉（小学/初中/高中分组），如「三年级」 |
| 班级数量 | | 0–40；**0 = 仅建年级**，≥1 按「三(1)班」规则批量生成 |

年级名称、排序、`grade_code` 自动生成；一行文案预览将创建的内容。开学时对每个年级各操作一次，或使用批量导入。

### 成员（右侧）

| 能力 | 说明 |
|------|------|
| 添加成员 | 弹窗：姓名、手机号、初始密码、角色、**所属年级**（年级组长/班主任）、**所属班级**（班主任）、**账号状态（单选）**；岗位范围由所选年级/班确定；选项来自组织树，无则先在左侧新建；可预填当前树选中项 |
| 编辑 | 行内「编辑」：改角色、岗位范围、**账号状态（单选：启用/禁用，非下拉）** |
| 批量导入 | 可导入年级班级结构或成员岗位（模板分 sheet） |
| 不做 | 短信邀请开通（本期） |

### 新建班级（简化）

| 字段 | 必填 | 说明 |
|------|------|------|
| 年级 | — | 只读，来自左侧选中年级 |
| 班号 | ✓ | 同年级唯一；自动建议下一号 |
| 班级名称 | — | 根据班号自动生成，如三(3)班 |

### 编码规则（演示）

- `grade_code`：`GR-{级别码}`，如 `GR-PRIMARY-G3`
- `class_code`：由 `grade_code` 派生 + 班号，如 `CLS-PRIMARY-G3-C1`

### 重命名 / 删除（与业务联动）

正式规则见 PRD **[4.8.3 年级/班级重命名与删除](../PRD/学校端/学校端功能清单.md#483-年级--班级重命名与删除业务规则)**。摘要：

| 操作 | 要点 |
|------|------|
| **重命名** | `grade_id` / `class_id` **不变**；成员 scope、学生档案 **自动跟新名**；投保订单 **快照不改** |
| **删除（撤销）** | 节点 `archived`；**不删**档案与订单；有档案或「仅辖本班」成员时 **阻断** 或先迁移 |

当前 HTML 原型删除仅为 DOM 移除 + 简单 confirm，**未实现**预检阻断；实现以对 PRD 为准。

## B 端 UI 规范（原型样式基线）

参考：[设计师必备！B端网页设计参数大全](https://www.uisdc.com/b-end-ui-specs)（优设 · 酸梅干超人，2026）

| 类别 | 参数 | 本原型采用 |
|------|------|------------|
| 画布 | 最佳 1440×900，最小 1280×800 | 主内容区 `max-width` + 侧栏 240px |
| 页边距 | 12–40px | `--page-margin: 24px` |
| 正文 | 13–16px | 14px |
| 标题 | 13–24px | H2 20px / H1 24px |
| 行高 | 1.2–1.8× 字号 | 正文约 1.57（22/14） |
| 按钮高度 | 常规 28–36px | 32px（大按钮 36px） |
| 输入框 | 与按钮对齐 | 32px |
| 圆角 | B 端偏克制 | 4–8px（卡片 8px） |

**样式文件**

| 文件 | 用途 |
|------|------|
| `b-end-tokens.css` / `b-end-responsive.css` / `shared-ui.css` / `b-end-pager.*` | 本目录 B 端 UI 基线与分页（学校端、平台端各一份） |
| `学平险-学校端-sidebar.css` | 学校端侧栏 |
| `学平险-学校端-b-end.css` | 学校端页面覆盖（置于各页 `</style>` 之后） |
| `学平险-学校端-login-b-end.css` | 登录页 |

学校端带侧栏页面在 `</head>` 末尾依次引入：`shared-ui.css` → `学平险-学校端-sidebar.css` → `学平险-学校端-b-end.css`。

### 分辨率自适应（`b-end-responsive.css`）

| 视口宽度 | 场景 | 行为 |
|----------|------|------|
| ≥1920px | 大屏 | 页边距 32px；主内容子块 `max-width: 1680px` 避免行过长 |
| 1440–1919px | 最佳 | 默认侧栏 240px、边距约 24px |
| 1280–1439px | 最小笔记本 | 侧栏 220px、边距 16px；统计卡片列宽收紧 |
| 1024–1279px | 紧凑 | 侧栏 72px 图标模式，悬停可展开全文 |
| ≤1023px | 平板 | 左右分栏改单列；工具栏纵向铺满 |
| ≤767px | 小屏 | 统计 2 列；弹窗贴边；表格单元格收紧 |

主区域使用 `flex: 1; min-width: 0`，宽表格在卡片内横向滚动，不再强制整页 `min-width: 1280px`。

**平台端**：各页在 `shared-ui.css` 后增加 `平台端-b-end.css`。大屏下 `.main-content .page` 限宽居中；`layout-2` / `quick-grid` / 权限矩阵行随宽度折叠。

### 表格分页（`b-end-pager.js`）

列表页表格下方统一分页条：**共 N 条**、每页条数（10/20/50）、上一页/页码/下一页、跳转页。

| 用法 | 说明 |
|------|------|
| 静态表格 | 在 `</table>` 后插入带 `data-b-end-pager` 的 `.pager` 容器（如 `data-page-size="10"`），页末引入 `b-end-pager.js` 并执行 `BEndPager.initAll()` |
| 动态渲染 | `BEndPager.create({ container, total, pageSize, current, onChange })`（学生档案、批量投保、续保管理） |

样式见 `b-end-pager.css`（已由 `shared-ui.css` 引入）。
